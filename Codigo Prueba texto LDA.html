Metodologia CRISP-DM ¶

El proceso estndar de la industria cruzada para la minera de datos (CRISP-DM) es un modelo de proceso que sirve como base para un proceso de ciencia de datos. Tiene seis fases secuenciales:

Comprensin empresarial: Qu necesita la empresa?

Comprensin de datos: Qu datos tenemos/necesitamos? Est limpio?

Preparacin de datos: Cmo organizamos los datos para el modelado?

Modelado Qu tcnicas de modelado debemos aplicar?

Evaluacin: Qu modelo cumple mejor con los objetivos comerciales?

Implementacin: cmo acceden las partes interesadas a los resultados?

Creacion Funcion de Lematizacion ¶

La lematizacin es el proceso mediante el cual las palabras de un texto que pertenecen a un mismo paradigma flexivo o derivativo son llevadas a una forma normal que representa a toda la clase. Esta forma normal, llamada lema, es tpicamente la palabra utilizada como entrada en los diccionarios de lengua: el infinitivo para las conjugaciones verbales, el masculino singular para adjetivos, etc. link

Para crear la funcin de lematizacin, se utiliza un conjunto de datos, en los cuales se puede obtener la forma de la palabra, y su raz. Para descargar el conjunto de dato de lematizacin, en el siguiente link.

Creacion Funcion para la Limpieza de datos. ¶

  1. Primero se realiza un proceso de limpieza, en la cual se remueven de los tweets cosas como nmeros, caracteres especiales, emoticones puntos, comas y dems, y luego pasar todas las palabras a forma minscula.

  2. Posteriormente se realiza el proceso de tokenizacin, el cual consiste en separar palabra por palabra cada tweet, en una lista para cada tweet.

  3. Despus de esto, se remueven las stopwords (palabras no relevantes como el, las, con); adems de estas palabras StopWords en espaol, se pueden agregar ms palabras que no se deseen observar.

  4. Por ltimo, se realiza el proceso de lematizacin, para evitar la redundancia de palabras, y su significado.

def cleaner(word):
    #nlp = es_core_news_sm.load() # importar modulo 
    list_word_clean1 = []
    list_word_clean2 = []
    lemma = []
    bigram = []
    
    # 
    original = str(word)
    # remover links, arrobas, numerales y demas
    word = re.sub(r'((http|https)\:\/\/)?[a-zA-Z0-9\.\/\?\:@\-_=#]+\.([a-zA-Z]){2,6}([a-zA-Z0-9\.\&\/\?\:@\-_=#])*', '', str(word), flags=re.MULTILINE)
    # quitar signos de interrogacion
    word = re.sub(r'(?::|;|=)(?:-)?(?:\)|\(|D|P)', "", str(word))
    # quitar numerales
    word = re.sub(r'\#\.', '', str(word))
    # quitar separacion de texto
    word = re.sub(r'\n', '', str(word))
    # quitar comas
    word = re.sub(r',', '', str(word))
    word = re.sub(r'\-', ' ', str(word))
    # se quitan los tres puntos seguidos
    word = re.sub(r'\.{3}', ' ', str(word))
    # quitar si hay letras seguidas
    word = re.sub(r'a{2,}', 'a', str(word))
    word = re.sub(r'{2,}', '', str(word))
    word = re.sub(r'i{2,}', 'i', str(word))
    word = re.sub(r'ja{2,}', 'ja', str(word))
    
    # eliminar espacios restantes, si quedo alguno
    word  =  re.sub(' +', ' ', str(word))
    word  = word.strip()
    word  =  re.sub('\r', '', str(word))
    
    
    # pasar todas las letras a minusculas, y eliminar stopwords en espaol
    for w1 in word.split(" "):
      if  w1.lower() not in spanish_stopwords:
        list_word_clean1.append(w1.lower())
        

    # pasar todas las letras a minusculas, y eliminar stopwords en ingles    
    for w2 in list_word_clean1:
      if  w2.lower() not in english_stopwords:
        list_word_clean2.append(w2.lower())

    # se une el texto resultante    
    out_text = " ".join(list_word_clean2)
    
    
    # lematizacion
    for a in list_word_clean2:
        if (a not in spanish_stopwords) and (a not in english_stopwords):
            lemma.append(lemmatize(a))
        # lemma1 = list(filter(None, lemma))

    # se encuentran todas las palabras que empiecen por @ para mencionar usuarios
    at = re.findall(r'[@]\S*', str(original))
    # se encuentran todas las palabras que empiecen por # para los hashtags
    hashtags = re.findall(r'[#]\S*', str(original))
  
    # la funcion devuelve las frases tratadas, la lista de palabras, los arrobas, los hashtags
    return word, list_word_clean2,out_text , at,hashtags, lemma

Metrica de Coherencia de Topicos. ¶

En el siguiente bloque, se crea un funcion para calcular la metrica de coherencia de topicos. Entre mas alta la metrica, mejor.

Qu es la coherencia de topicos?

"Las medidas de coherencia de topicos califican un solo tema midiendo el grado de similitud semntica entre las palabras de alto puntaje en el tema. Estas medidas ayudan a distinguir entre temas que son temas interpretables semnticamente y temas que son artefactos de inferencia estadstica. Se dice que un conjunto de enunciados o hechos es coherente, si se apoyan entre s. Por lo tanto, un conjunto de hechos coherente puede interpretarse en un contexto que cubre todos o la mayora de los hechos." Fuente

# supporting function
def compute_coherence_values(corpus, dictionary, k):
    
    lda_model = gensim.models.LdaMulticore(corpus=corpus,
                                           id2word=id2word,
                                           num_topics=k, 
                                           random_state=124)
    
    coherence_model_lda = CoherenceModel(model=lda_model, texts=df2['word_lemma'], 
                                         dictionary=id2word, coherence='c_v')
    
    return coherence_model_lda.get_coherence()

import tqdm # esta libreria ayuda a saber el progreso de cualquier proceso

grid = {}
grid['Validation_Set'] = {}

# Rango de Topicos
min_topics = 2
max_topics = 30
step_size = 1
topics_range = range(min_topics, max_topics, step_size)


# Set de validacion
num_of_docs = len(corpus)
corpus_sets = [gensim.utils.ClippedCorpus(corpus, int(num_of_docs*0.75)), 
               corpus]

corpus_title = ['75% Corpus', '100% Corpus']

model_results = {'Validation_Set': [],
                 'Topics': [],
                 'Coherence': []
                }

# Puede tomar un largo tiempo en correr
if 1 == 1:
    pbar = tqdm.tqdm(total=(len(topics_range)*len(corpus_title)))
    
    # iterar a travs de corpus de validacin
    for i in range(len(corpus_sets)):
        # iterar a travs de la cantidad de temas
        for k in topics_range:
            # obtener la puntuacin de coherencia para los parmetros dados
            cv = compute_coherence_values(corpus=corpus_sets[i], dictionary=id2word, 
                                                  k=k)
            # Guardar los resultados del modelo
            model_results['Validation_Set'].append(corpus_title[i])
            model_results['Topics'].append(k)
            model_results['Coherence'].append(cv)
                    
            pbar.update(1)
    print(pd.DataFrame(model_results))#.to_csv('lda_tuning_results2.csv', index=False)
    pbar.close()

Modelado: Modelo LDA.   ¶
La Asignacin de Dirichlet Latente (LDA por sus siglas en ingles), es uno de los algoritmos generativos ms usados y con mejor desempeo en cuanto a modelado de tpicos.


Este algoritmo tiene sus bases probabilsticas en la estadstica bayesiana e inferencia estadstica bayesiana, pero que gracias a su gran flexibilidad, es fcilmente implementable en lenguaje de maquina y al igual que el algoritmo de Bayes Ingenuo, su asuncin de independencia facilita la computacin.


En la Asignacin de Dirichlet Latente, se considera que cada documento, o bolsa de palabras, tiene una mezcla de varios temas que son asignados a travs de este algoritmo; en esta parte se supone que la distribucin de cada tema proviene de
una distribucin de probabilidad de Dirichlet.

Interpretacion de Resultados.   ¶














Se puede observar que las palabras que mas aparecen son davivienda y respuesta.


Se puede interpretar, que los usuarios que acuden a la red social Twitter, y mencionan al banco Davivienda, lo hacen principalmente con la intencin de plantear un requerimiento, y buscando una respuesta. Se puede observar esto en los primeros tpicos que se forman al momento de imprimir el resultado de tpicos, del modelo LDA.


Tambin se puede observar que la palabra privado aparece tambin en el primer tpico formado por el modelo, lo que hara referencia a que por favor se envi el requerimiento por un mensaje privado.


En cuanto al segundo tpico, se puede observar que tiene la palabra daviplata, cuenta, problema, lo cual se puede de interpretar con que varios usuarios estn presentando inconvenientes con la plataforma de Daviplata.
